# Module Acknowledgements

This guidance covers implementing acknowledgement messages in RISC OS modules. These messages are displayed by the Task Manager to credit authors and document licenses.

## Workflow

1.  **Select Acknowledgement Group**: Determine the appropriate group for your module.
    - `OS`: Core Operating System components.
    - `System`: System-level modules and extensions.
    - `Tertiary`: Standard third-party modules (most common).
    - `Commercial`: Paid applications or proprietary modules.
    - `Freeware`: Components distributed for free.
    - `User`: Personal or user-provided components.
2.  **Implement Logic**: Create files to handle the service call using the implementation template below.
3.  **Register Service**: Add `Service_TaskManagerAcknowledgements` to your CMHG file.
4.  **Integrate with Lifecycle**: Call the announce function during module initialization and finalization.

## Definitions

The following constants are used for the Task Manager acknowledgements service:

- `Service_TaskManagerAcknowledgements`: `0x42680`

### Acknowledgement Groups

| Constant | Value | Description |
| :--- | :--- | :--- |
| `TaskManager_AckGroup_OS` | `0x0000` | Operating System components |
| `TaskManager_AckGroup_System` | `0x1000` | System-level modules |
| `TaskManager_AckGroup_Tertiary` | `0x2000` | Third-party modules (standard) |
| `TaskManager_AckGroup_Commercial` | `0x3000` | Commercial applications/modules |
| `TaskManager_AckGroup_Freeware` | `0x4000` | Freeware components |
| `TaskManager_AckGroup_User` | `0x5000` | User-provided components |

## Implementation Template

### Header File (`h/acknowledgements`)

```c
#ifndef ACKNOWLEDGEMENTS_H
#define ACKNOWLEDGEMENTS_H

#include <stdbool.h>
#include "swis.h"

void acknowledgements_announce(bool startup);
void acknowledgements_service(int service_number, _kernel_swi_regs *r, void *pw);

#endif
```

### Source File (`c/acknowledgements`)

```c
#include <stdbool.h>
#include <stdlib.h>
#include "swis.h"
#include "modhead.h"

/* Define local constants if not in Switcher.h */
#ifndef Service_TaskManagerAcknowledgements
#define Service_TaskManagerAcknowledgements 0x42680
#endif
#ifndef TaskManager_AckGroup_Tertiary
#define TaskManager_AckGroup_Tertiary 0x2000
#endif

static const char Acknowledgements_String[] =
    "My Module: Copyright (c) 2026, My Name\n"
    "Additional components: Copyright info";

void acknowledgements_announce(bool startup)
{
    /* Let TaskManager know acknowledgements have changed */
    _swix(OS_ServiceCall, _INR(1,2), Service_TaskManagerAcknowledgements, 0);
}

typedef void (*taskmanager_addmessages_func)(int group, const char *title, char *message);

void acknowledgements_service(int service_number, _kernel_swi_regs *r, void *pw)
{
    if (service_number == Service_TaskManagerAcknowledgements)
    {
        taskmanager_addmessages_func func = (taskmanager_addmessages_func)r->r[2];
        if (func != NULL)
        {
            /* Use TaskManager_AckGroup_Tertiary for standard third-party modules */
            func(TaskManager_AckGroup_Tertiary, Module_Title, (char *)Acknowledgements_String);
        }
    }
}
```

## Integration

### 1. CMHG File
Ensure the service is registered in your `.cmhg` file:
```cmhg
service-call-handler: Mod_Service Service_TaskManagerAcknowledgements
```

### 2. Module Lifecycle
Call the announcement in `Mod_Init` and `Mod_Final`:
```c
/* Mod_Init */
acknowledgements_announce(true);

/* Mod_Final */
acknowledgements_announce(false);
```

### 3. Service Handler
Dispatch to your acknowledgement handler in `Mod_Service`:
```c
void Mod_Service(int service_number, _kernel_swi_regs *r, void *pw)
{
    acknowledgements_service(service_number, r, pw);
}
```

## Best Practices

-   **Concise Strings**: Keep acknowledgement strings clear and formatted with newlines if necessary.
-   **Copyright Info**: Include the copyright year and the author's name or organization.
-   **Module Title**: Use the `Module_Title` macro (generated by CMHG) to ensure consistency.
